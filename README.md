## Overview

This is a simple Bash tool that automatically backs up, compresses and archives Linux log files. It is designed to be easy to use, and works on most Linux distributions. Ideal for users who want to keep their log files organized.

## ‚ö° Features 

- Copies all log files from '/var/log'
- Compresses them into a '.tar.gz' archive
- Archives are named with date and time
- Desktop notifications via 'notify-send'
- Can be scheduled to run automatically using cron
- Safe deletion of old backups

## üì¶ Dependencies

- 'cron' (or 'cronie' on some distros)
- 'tar'
- 'bash'
- 'notify-send'

## üöÄ Installation

1. Install dependencies: 
```bash
sudo apt install cron              # Debian
```
```bash
sudo dnf install cronie            # Fedora
```
```bash
sudo pacman -S cronie              # Arch 
```
---

  

2. Enable and start cron/cronie: 

```bash
sudo systemctl enable cron
```
```bash
sudo systemctl start cron
```


---


3. Clone the repository: 

```bash
git clone https://github.com/edimurg/log_backup_tool.git
```
```bash
cd Log_Backup_tool
```

---

4. Allow permissions:
   
```bash
chmod +x log_backup_tool.sh
```

---

5. Run the tool:

```bash
./log_backup_tool.sh setup
```
---
## üìù Usage

``` bash
./log_backup_tool.sh setup    # Enter the setup mode
```
```bash
./log_backup_tool.sh backup   # Run a backup immediately
```
```bash
./log_backup_tool.sh delete   # Delete all old backups
```
---
## ‚ö†Ô∏è Limitations

- By default, the script expects to be located in the home directoty (~/).
- Needs sudo to access '/var/log'.
- Works only on Linux systems with cron.
- Large logs may take time to compress.
- You might need to put 'SHELL=/bin/bash' in the beginning of the crontab file (crontab -e)
