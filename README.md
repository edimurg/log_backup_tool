A simple Bash script that saves, compresses, and archives Linux log files automatically, at scheduled interals using **cron**.


## âš¡ Features 

- Copies all log files from '/var/log'
- Compresses them into a '.tar.gz' archive
- Archives are named with the current date and time
- Desktop notifications via 'notify-send'
- Can be scheduled to run automatically using cron

## ðŸ“¦ Dependencies

- 'cron' (or 'cronie' on some distros)
- 'tar'
- 'bash'
- 'notify-send'

## ðŸš€ Instalation

1. Install dependencies: 
```bash
sudo apt install cron              # Debian
```
```bash
sudo dnf install cronie            # Fedora
```
```bash
sudo pacman -S cronie              # Arch 
```
---

  

2. Enable and start cron/cronie: 

```bash
sudo systemctl enable cron
```
```bash
sudo systemctl start cron
```


---


3. Clone the repository: 

```bash
git clone https://github.com/edimurg/log_backup_tool.git
```
```bash
cd Log_Backup_tool
```

---

4. Allow permissions:
```bash
chmod +x setup.sh
```
```bash
chmod +x log_backup_saver.sh
```

---

5. Run the setup.sh script:

```bash
./setup.sh
```
---
## âš ï¸ Limitations

- By default, the script expects to be located in the home directoty (~/).
- Needs sudo to access '/var/log'.
- Works only on Linux systems with cron.
- Large logs may take time to compress.
- You might need to put 'SHELL=/bin/bash' in the beginning of the crontab file (crontab -e)
